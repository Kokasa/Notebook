**1. 成员 (Member)**

- **定义：** 成员是指类（Class）中包含的变量（属性/字段）和方法（函数）。可以把类比作一个家庭，成员就是家庭中的人（变量）和他们做的事情（方法）。
    
- **例子：** 想象一个“家庭”类。
    
    - **变量（属性/字段）：**
        - 家庭成员数量（`numberOfMembers`）：比如 3 人。
        - 家庭地址（`address`）：比如“XX市XX路XX号”。
        - 家庭收入（`income`）：比如每月 10000 元。
    - **方法（函数）：**
        - 做饭（`cook()`）：家庭成员可以做饭。
        - 打扫卫生（`clean()`）：家庭成员可以打扫卫生。
        - 赚钱（`earnMoney()`）：家庭成员可以通过工作赚钱。
    
    这些“家庭成员数量”、“家庭地址”、“做饭”等都是“家庭”类的成员。
    

**2. 静态 (Static)**

- **定义：** `static` 关键字用于修饰成员（变量和方法），表示这些成员属于类本身，而不是类的某个具体实例（对象）。可以把类比作“公共财产”，所有家庭都可以共享使用。
    
- **例子：** 在上面的“家庭”类中，如果有一个“地球人口总数”的变量，它显然不属于某个特定的家庭，而是属于整个人类（所有家庭共享）。这时就可以用 `static` 来修饰：
    
    Java
    
    ```
    class Family {
        static long worldPopulation = 7800000000L; // 地球人口总数，所有家庭共享
    
        // ... 其他成员
    }
    ```
    
    - **静态变量：** 所有该类的对象共享同一个静态变量。修改其中一个对象的静态变量，其他对象的该静态变量也会受到影响。
    - **静态方法：** 可以通过类名直接调用，而不需要创建类的实例。例如 `Math.sqrt()` 是一个静态方法，可以直接使用 `Math.sqrt(9)` 调用，而不需要先创建一个 `Math` 对象。
- **与实例相关的对比：** 非静态的成员（也称为实例成员）属于类的每个具体实例。每个家庭都有自己的家庭地址和收入，这些是实例相关的。
    

**3. 私有 (Private)**

- **定义：** `private` 关键字用于修饰成员（变量和方法），表示这些成员只能在当前类内部访问，其他类无法直接访问。可以比作家庭内部的“隐私”，外人无法得知。
    
- **例子：** 在“家庭”类中，家庭收入（`income`）通常是隐私，不希望被外人知道，这时就可以用 `private` 来修饰：
    
    Java
    
    ```
    class Family {
        private int income; // 家庭收入，只能在 Family 类内部访问
    
        public int getIncome() { // 提供一个公共方法来获取收入
            return income;
        }
    
        public void setIncome(int income) { // 提供一个公共方法来设置收入
            this.income = income;
        }
    
        // ... 其他成员
    }
    ```
    
    其他类只能通过 `getIncome()` 和 `setIncome()` 这样的公共方法来间接访问和修改 `income`，而不能直接访问。这体现了封装性，保护了数据的安全性。
    

**总结**

| 特性  | 成员 (Member)       | 静态 (Static) | 私有 (Private) |
| --- | ----------------- | ----------- | ------------ |
| 归属  | 属于类或类的实例          | 属于类本身       | 属于类或类的实例     |
| 访问  | 类的内部和外部（取决于访问修饰符） | 通过类名直接访问    | 只能在类的内部访问    |
| 共享  | 每个实例拥有各自的副本       | 所有实例共享同一个副本 | 每个实例拥有各自的副本  |
| 比喻  | 家庭成员              | 公共财产        | 家庭隐私         |
